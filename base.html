RISC-V Instruction Set Emulator

Use C language to create an emulator that is able to execute a subset of the RISC-V ISA. Provide the machinery to decode and execute a couple dozen RISC-V instructions. When given an instruction encoded as a 32-bit integer, the emulator can reproduce the original RISC-V instruction in human- readable format. It can print the instruction name; print all arguments, following the order and formatting; fetch an instruction from memory, using the PC as the address; executes the instruction and updates the PC. In addition, I have to modify the appropriate registers; make any necessary calls to memory; update the program counter to refer to the next instruction to execute; takes an address, a size, and a value and stores the first (and next)-size- bytes of the given value at the given address.

A small version of database

Build a small version of what is called a relational database management system (DBMS), as well as a DSL (Domain Specific Language) with which a user can interact with your database. The language implemented is similar to the declarative programming language SQL, which is used in the real world to interact with and query databases, large or small. First, we make our database support some commands, which is very similar to SQL. Then, this database can handle special values, like NaN and Novalue. It is capable of parsing the commands your database should accept and displaying a brief summary of the command. Finally, it can join multiple tables together by following the rule of Cartesian Product of the tables.

A Scheme Interpreter

Use python to develop an interpreter for a subset of the Scheme language. Basically, it contains: reading Scheme expressions; symbol evaluation; calling built-in procedures; definitions; lambda expressions and procedure definition; calling user-defined procedures; evaluation of special forms.
